What this repo family really is

These twelve repositories are not twelve unrelated prototypes. They represent a single evolving idea — a regional-language learning platform that teaches through Hindi or English as the bridge language — explored across ten expert-led iterations (Indore01–Indore10), one umbrella brand repo (IndiLingo), and one fully-built application (IndiLingo).

The pattern is clear: each Indore0x repo pushed one capability forward — better backend, richer gamification, offline state, expanded curriculum — while the umbrella name IndiLingo held the overall product vision. Indore10 is the most recent iteration and introduces the most complete curriculum model seen in the series.

Core recommendation (unchanged from v1, now confirmed by the build)

Keep IndiLingo as the brand and umbrella name. Mine each repo for its strongest capability. The IndiLingo app already materializes this strategy — it is the living proof that the merger approach works.

The twelve repos are not competing drafts. They are chapters of a single story. The story now has a working product.

What changed in v2 — Indore10 and beyond

New in this revision

v1 of this blueprint covered nine Indore0x repos plus IndiLingo. This v2 adds Indore10 (the latest iteration) and the IndiLingo built app as canonical outputs, and updates the merge guidance accordingly.

Indore10 — what it adds

Indore10 is the tenth and most recent expert iteration. It consolidates the curriculum model from Indore09, layers in the dual-language bridge approach more explicitly, and formalises the exercise type taxonomy that the IndiLingo build ultimately implemented. Its key additions over Indore09:

  • Formalised exercise taxonomy — five types explicitly named: script_practice, multiple_choice, translate, fill_blank, match_pairs.
  • Script-first curriculum ordering — the principle that learners must complete a "Script & Alphabet" unit before any vocabulary content. No more assuming the learner can already read the script.
  • 15-language scope — moves from 7–9 languages to a full 15, covering all constitutionally recognised Indian languages plus Nepali and Sindhi.
  • Spaced repetition hooks — a exercise_mistakes data model described in design docs, which became the review queue in the final build.

What the IndiLingo build confirmed

The IndiLingo application built from this blueprint validated all the merger recommendations. Every major capability from the twelve repos has been successfully combined into one product:

Backend & API

OpenAPI-first contract, Drizzle ORM, PostgreSQL, Express 5. Drawn from Indore02.

Curriculum Engine

15 languages × 4 units × 5 exercise types, script-first ordering. Drawn from Indore09 + Indore10.

Gamification

XP, streaks, 3-star lesson rating, podium leaderboard, 5 achievement badges. Drawn from Indore04 + Indore08.

Spaced Repetition

Mistake tracking table, review queue ordered by missedCount, master-on-correct deletion. Drawn from Indore08 + Indore10.

Exercise types (5)

  • script_practice — Show the native-script character at large size, pick the correct romanization from 4 options (or vice versa). First exposure to each script.
  • multiple_choice — Show a question in Hindi/English, pick correct translation from 4 options.
  • translate — Show a phrase, select the correct full translation.
  • fill_blank — Sentence with a visual gap; tap to fill from 4 option buttons.
  • match_pairs — Two-column grid; tap to match left-column questions with right-column answers.

Gamification

  • XP earned per lesson completion (server-authoritative, idempotent)
  • 1–3 star rating based on accuracy (≥50% → 1 star, ≥70% → 2, 100% → 3)
  • Daily streak tracking with automatic increment/break logic
  • Podium leaderboard — top 3 on medal-tiered columns (2nd left, 1st elevated centre, 3rd right)
  • 5 achievement badges: First Steps, Century Club, On Fire, Rocket Learner, Champion

Spaced repetition review

  • exercise_mistakes table tracks per-user, per-exercise miss counts
  • Review queue ordered by missedCount DESC, lastMissedAt ASC, limit 10
  • Correct answer on review calls master endpoint → row deleted from queue
  • Wrong answer increments missedCount via upsert

What comes next — three natural additions

Expo mobile app

An artifacts/mobile/ Expo app using the same API server and OpenAPI contracts. This is the natural follow-on from Indore03/08's mobile work.

Admin dashboard

An artifacts/admin/ React app for curriculum management, user analytics, and lesson content editing. Draws on Indore05/07's admin surface ideas.

TTS integration

Adds audio pronunciation to every exercise, starting with the script_practice type. Directly implements Indore01's long-standing TTS roadmap item.

Merge and evolution roadmap

The original v1 merge sequence has been completed. The roadmap below is the post-merge evolution sequence — what to do next now that the foundation is running.

  1. [Done] Establish the monorepo with API, DB, and OpenAPI contracts from Indore02.
  2. [Done] Build the web lesson player drawing on Indore03's interaction model.
  3. [Done] Add gamification (XP, streaks, podium leaderboard, badges) from Indore04/08.
  4. [Done] Seed 15 languages with script-first curriculum from Indore09/10.
  5. [Done] Implement spaced repetition review from Indore08/10 data model.
  6. [Next] Build the Expo mobile app (apps/mobile) using Indore03/08's native layer concepts. The API is already ready — the mobile client just needs to be built.
  7. [Next] Add TTS audio to script_practice exercises — Indore01's most practical contribution.
  8. [Later] Build the admin/analytics dashboard from Indore05/07's surface ideas.
  9. [Later] Expand the curriculum with community contribution tools, more exercise types, and deeper content per language.

The v1 recommendation was "identify the strongest owner for each capability, then consolidate around one product vision." That work is done. The platform now exists. The question for v2 is: which new capability should come next?
